# Launch Hardening — TeamFrame Pre-Launch Reference

This folder contains all artifacts related to the pre-launch hardening sprint. It covers the consolidated audit findings from five separate reviews, the batched execution plan, accepted risks register, operational runbooks, verification checklists, and the operational readiness checklist that must be completed before public launch.

- [`audit-findings-consolidated.md`](audit-findings-consolidated.md) — Synthesized findings from all five audits with status tracking
- [`hardening-execution-plan.md`](hardening-execution-plan.md) — Three-batch Cursor execution plan with verification steps
- [`accepted-risks.md`](accepted-risks.md) — Consciously deferred items with reasoning and revisit triggers
- [`operational-readiness-checklist.md`](operational-readiness-checklist.md) — Pre-launch operational checklist
- [`cursor-batch-prompts/`](cursor-batch-prompts/) — Batched Cursor prompts for each hardening sprint batch
- [`runbooks/`](runbooks/) — Operational runbooks for support, rollback, incident response, secret rotation
- [`audit-reports-archive/`](audit-reports-archive/) — Immutable archive of the five original audit reports
- [`verification/`](verification/) — Verification checklists and smoke test records
